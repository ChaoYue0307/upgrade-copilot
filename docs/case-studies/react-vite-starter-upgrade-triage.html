Case study

React/Vite starter upgrade triage.

A modern frontend starter shows how upgrade risk can come from tooling coordination, not only old framework versions.

Readiness: yellow
  • Repo: CodingGarden/react-ts-starter
  • Stars at review: 214
  • Primary risk: Vite, Vitest, TypeScript, ESLint, and React changes bundled together
  • Best first action: preserve baseline typecheck, lint, test, and build
Evidence

What the repository shows.

Vite gapVite 5.4.8 versus current Vite 8.0.12.
Test gapVitest 2.1.2 with jsdom and setup files versus current Vitest 4.x.
Lint gapESLint 8 and @typescript-eslint 7 need a focused config migration.
React gapReact 18.3.1 can move to React 19 after tooling is stable.
Router gapReact Router 6 to 7 should be validated as app behavior.
CI uncertaintyNo visible .github/workflows directory was found.
Recommended plan

Keep tooling noise away from runtime changes.

1
Baseline validation PR

Run typecheck, lint, test, and build. Add CI if this starter is maintained.

2
Vite/Vitest batch

Upgrade Vite, Vitest, @vitejs/plugin-react, and jsdom together, then validate tests and builds.

3
ESLint/TypeScript batch

Upgrade TypeScript, ESLint, and @typescript-eslint separately from runtime packages.

4
Router migration

Upgrade React Router separately and validate route behavior.

5
React 19 migration

Upgrade React after tooling is stable and validate Testing Library behavior.
