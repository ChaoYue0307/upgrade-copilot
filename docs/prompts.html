Copy-paste prompts for risky upgrade work.

Use these in a Codex thread after installing Upgrade Copilot. They are designed for real upgrade moments: triage, breaking changes, CI failures, PR splitting, and team planning.

Find safe dependency batches

Use when a repo has many outdated packages and you need a low-risk starting point.

Use Upgrade Copilot to inspect this repo's package manifests, lockfiles, tests, and CI. Find the safest dependency upgrades, group them into small PR batches, and list validation commands for each batch.

Map framework breaking changes

Use before upgrading Next.js, React, Rails, Django, Terraform, Kubernetes, or another major framework.

Use Upgrade Copilot to map breaking changes for upgrading this framework. Connect each relevant upstream change to local files, configs, tests, and runtime behavior. Ignore changes that do not appear relevant to this repo.

Diagnose CI after an upgrade

Use when a dependency bump caused tests, type checks, lint, or build steps to fail.

Use Upgrade Copilot to diagnose this upgrade-related CI failure. Start from the first meaningful error, classify the failure, compare it with the upgrade diff, and propose the smallest fix plus the validation command to rerun.

Split a migration into PRs

Use when a migration is too large or risky for one review.

Use Upgrade Copilot to split this migration into reviewable PRs. Separate dependency bumps, config changes, mechanical rewrites, behavior changes, tests, and cleanup. Include merge order, validation commands, and rollback notes.

Create a team upgrade backlog

Use when you want a roadmap that managers and tech leads can fund.

Use Upgrade Copilot to build a team upgrade backlog for this repository. Rank work by risk, urgency, engineering effort, validation confidence, and rollback difficulty. Return a 30/60/90-day plan.
